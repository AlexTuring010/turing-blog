Όλα ξεκίνησαν με την εξής σκέψη: θέλω να καταλάβω πραγματικά τι κάνει ο compiler.

Όχι "καταλαβαίνω" όπως καταλαβαίνεις τις λέξεις σε ένα textbook. Καταλαβαίνω όπως όταν έχεις γράψει τον δικό σου και ξέρεις γιατί κάθε γραμμή κώδικα είναι εκεί που είναι.

01Τι κάνει ο μ-compiler

Σήμερα, ο compiler δέχεται μια απλή functional γλώσσα με integers, booleans, if-expressions, και first-class functions. Παράγει LLVM IR, που μετά μετατρέπεται σε native binary.

Δεν έχει: type system (έρχεται), pattern matching, generics, modules, ή error messages που δεν είναι τραγικά.

02Αρχιτεκτονική

Τέσσερα phases, σχεδόν παραδοσιακά:

Lexer → Παράγει tokens. Γραμμένος με το hand, χωρίς generators. Επειδή ήθελα να δω πώς γίνεται.

Parser → Recursive descent. Δοκίμασα πρώτα Menhir (parser generator για OCaml) αλλά γύρισα σε hand-written για debugging visibility.

Type checker → Hindley-Milner inference. Αυτό ήταν το πιο δύσκολο part — έπρεπε να ξαναδιαβάσω το σχετικό κεφάλαιο του Crafting Interpreters τρεις φορές.

Codegen → Emits LLVM IR via το OCaml binding του LLVM. Το LLVM κάνει την heavy work (optimization, register allocation, codegen).

Το repo έχει περισσότερες λεπτομέρειες — ο κώδικας είναι σχολιασμένος γιατί κυρίως τον γράφω για να μαθαίνω, όχι για production.
